งานที่สอง · Estimating E_out

ความเอนเอียง (bias) และความแปรปรวน (variance)
ของ “ตัวประมาณค่า E_out” 3 แบบ

จำลอง (simulation) เพื่อดูว่าแต่ละวิธีประมาณ error นอกกลุ่มตัวอย่างได้ แม่น (bias ต่ำ) และ นิ่ง (variance ต่ำ) แค่ไหน — ปรับค่าบนแถบด้านบนแล้วกราฟทั้งหน้าจะคำนวณใหม่สด ๆ

ฟังก์ชันเป้าหมาย

f(x) = sin(π·x)

สุ่ม x ~ U[−1, 1] จำนวน n จุด
y = f(x) + noise, noise ~ N(0, σ²)

แบบจำลอง 2 แบบ

  • constant — เส้นแนวนอน h(x)=b (ค่าเฉลี่ยของ y)
  • linear — เส้นตรง h(x)=a·x+b (least squares)

ดึงมาจากงานแรกผ่าน models.py

ตัวประมาณค่า E_out 3 แบบ

  • resub วัด error บนข้อมูลที่เทรนไปแล้ว
  • holdout แบ่ง train/test
  • k-fold วนทดสอบ k ก้อนแล้วเฉลี่ย
แนวคิดสำคัญ

นิยามที่ใช้วัด “ความแม่น” ของตัวประมาณ

E_out จริง (ค่าอ้างอิง)

E_out(D) = Ex[ (g(x) − f(x))² ] + σ²
  • พจน์แรก = error เฉลี่ยของโมเดลเทียบ f จริง (ประมาณด้วยกริดถี่ 1001 จุด)
  • พจน์ σ² = noise ที่ลดไม่ได้ (irreducible error)
  • ใช้โมเดลที่เทรนบน “ข้อมูลเต็มชุด” เป็นตัวอ้างอิง

bias / variance / MSE ของตัวประมาณ

bias = mean(ค่าประมาณ) − mean(E_out จริง) variance = Var(ค่าประมาณ) MSE = mean( (ค่าประมาณ − E_out จริง)² ) ≈ bias² + variance

คำนวณจากการทำซ้ำ M ชุดข้อมูล (Monte Carlo). bias เป็นลบ = ประมาณต่ำเกินจริง (มองโลกแง่ดี)

ทุกกราฟ/ตารางด้านล่างคำนวณด้วย JavaScript ในเบราว์เซอร์คุณเอง (พอร์ตตรง ๆ จาก perf_estimators.py) — ไม่ต้องต่อเน็ต ปรับค่าแล้วเห็นผลทันที

หัวข้อ (c)

เปลี่ยนสัดส่วน holdout และจำนวน fold k

ใช้ model = linear กวาดค่า ratio และ k เพื่อดูว่าแบบใดให้ค่าประมาณ “นิ่ง” (variance ต่ำ) กว่า. ส่วนนี้คำนวณหนัก จึงกดปุ่มรันเอง

holdout: ค่าประมาณเฉลี่ย ± SD vs test ratio

k-fold: ค่าประมาณเฉลี่ย ± SD vs จำนวน fold

แถบ = ±1 SD (ยิ่งแคบ = เที่ยงตรง/Precision สูง) เส้นทึบ = ค่าเฉลี่ยประมาณ เส้นประ = E_out จริง (ระยะห่าง = bias)
ratiobiasvariance
kbiasvariance
แนวโน้ม: test set ใหญ่ขึ้น / k มากขึ้น → variance มักลดลง. k-fold (โดยเฉพาะ k=5–10) ให้ค่าประมาณแม่น/นิ่งกว่า holdout
หัวข้อ (d)

ปรับ n และ σ

ใช้ model = linear กวาด n = [10, 20, 50, 100] × σ = [0.1, 0.3, 0.5] ดูผลต่อ variance ของ k-fold และ bias ของ resub

k-fold: variance ของค่าประมาณ vs n (เส้นละ σ)

nσresub biasholdout varkfold var
แนวโน้ม: n มากขึ้น → variance ทุกวิธีลดลง (ข้อมูลเยอะ ประมาณนิ่งขึ้น); σ มากขึ้น → variance เพิ่ม และ resub ยิ่งมองโลกแง่ดี (bias ติดลบมากขึ้น)

คำตอบท้าย Lab

สรุปคำตอบตามที่โจทย์ถาม (อ้างตัวเลขจริง)

Lab 1 — Holdout

  • สัดส่วนที่เที่ยงตรงสุด: 0.5 (SD 0.72 ต่ำสุด)
  • train 90% เหวี่ยงเพราะ test เหลือ 2 จุด
  • train 10% พุ่งเพราะ train แค่ 2 จุด โมเดลพัง

Lab 2 — k-fold

  • k เที่ยงตรงสุด: 10-fold (SD 0.16)
  • Precision: CV > Holdout (0.16 vs 0.72)
  • Accuracy: ใกล้เคียงกัน (AVG ~7.5 ทั้งคู่)

Lab 3 — เพิ่ม n

  • n ↑ → SD ↓ มาก (เที่ยงตรงขึ้น)
  • AVG นิ่ง ~5.9 (แม่นยำขึ้น)
  • ข้อมูลเยอะช่วยทั้ง Precision และ Accuracy

Lab 4 — เทียบ 3 วิธี

  • แม่นยำ & เชื่อถือได้สุด: CV (ใกล้ 5.54 ไม่เอนเอียง)
  • resub เที่ยงตรงแต่มองโลกแง่ดี — ห้ามใช้จริง
  • holdout เที่ยงตรงแย่สุด (SD 2.84)

เลือกตัวประมาณอย่างไรดี?

resubstitution

เอนเอียงต่ำ (มองโลกแง่ดี) variance ต่ำ
✗ ห้ามใช้ประเมินสมรรถนะจริง

holdout

แทบไม่เอนเอียง แต่ variance สูง — ขึ้นกับว่าแบ่ง/สุ่มได้ชุดไหน

k-fold CV

ไม่เอนเอียงมาก และ variance ต่ำกว่า holdout
✓ ตัวเลือกที่ดีที่สุด

ยิ่งข้อมูลน้อย / noise มาก / โมเดลยืดหยุ่นสูง → การประมาณ E_out ยิ่งยากและแปรปรวนมากขึ้น
