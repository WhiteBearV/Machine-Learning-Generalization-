  (1) เปิด + โจทย์            [ 0:00 - 1:00 ]

(ชี้ที่จอ: หัวข้อสไลด์ + การ์ด 3 ใบหน้า intro)

"สวัสดีครับ วันนี้ผมจะพาไปดูคำถามง่ายๆ แต่สำคัญมากใน Machine Learning ครับ —
เวลาเราเทรนโมเดลเสร็จ เราจะรู้ได้ยังไงว่ามันเก่งจริงกับข้อมูลที่ไม่เคยเห็น?

ค่าที่เราอยากรู้เรียกว่า E_out หรือ error นอกกลุ่มตัวอย่าง ปัญหาคือเราวัดมัน
ตรงๆ ไม่ได้ ต้อง 'ประมาณ' เอา ซึ่งมีอยู่ 3 วิธีหลัก

คำถามของงานนี้คือ — 3 วิธีนั้น วิธีไหนประมาณได้ 'แม่น' และ 'นิ่ง' ที่สุด?
วันนี้ผมจะเน้นที่ 'การจำลอง' เป็นหลัก เพราะเป็นส่วนที่เห็นพฤติกรรมของทั้ง 3 วิธี
ได้ชัดที่สุด แล้วปิดท้ายด้วยการยืนยันด้วย Weka บนข้อมูลจริงแบบสั้นๆ ครับ"

  (2) แนวคิดหลัก             [ 1:00 - 2:30 ]
--------------------------------------------------------------------------------
(ชี้ที่จอ: การ์ด resub / holdout / kfold)

"เริ่มจาก 3 วิธีก่อนครับ
  - Resubstitution  — เทรนแล้ววัด error บนข้อมูลชุดเดิมที่เทรนไป
  - Holdout         — แบ่งข้อมูลเป็น train กับ test วัดบน test
  - k-fold CV       — แบ่งเป็น k ก้อน วนให้ทุกก้อนได้เป็น test แล้วเฉลี่ย

(ชี้ที่จอ: ตาราง Precision/Accuracy)

ทีนี้ 'แม่น' กับ 'นิ่ง' ที่ผมพูดถึง มันมีชื่อทางเทคนิคครับ
  - ความเที่ยงตรง (Precision) = ทำซ้ำแล้วได้ค่าใกล้เคียงกันไหม
        วัดด้วยค่าเบี่ยงเบนมาตรฐาน SD ยิ่งต่ำยิ่งเที่ยงตรง  ตรงกับ variance
  - ความแม่นยำ (Accuracy) = ค่าที่ได้ใกล้ความจริงไหม  ตรงกับ bias

เก็บสองคำนี้ไว้นะครับ เดี๋ยวเราจะใช้มันตัดสินทั้งงาน"

  (3) ผลจากการจำลอง (a)-(d)  [ 2:30 - 7:00 ]   << ส่วนหลัก เน้นตรงนี้ >>
--------------------------------------------------------------------------------

>> (a) ข้อมูลชุดเดียว   (~1:00)
(ชี้ที่จอ: ตาราง a)

"ทำไมต้องเริ่มจากการจำลอง? เพราะในการจำลอง เรา 'ตั้งความจริงเอง' ได้ครับ
ผมกำหนดฟังก์ชันจริงเป็น sin(pi x) เพราะฉะนั้นเรารู้เป๊ะๆ ว่า E_out จริงคือเท่าไร
พอรู้ค่าจริง เราถึงจะวัด bias ได้ตรงๆ ว่าแต่ละวิธีเดาพลาดไปเท่าไร ซึ่งบนข้อมูล
จริงเราทำแบบนี้ไม่ได้เลย

ทีนี้ดูข้อมูลชุดเดียวก่อน (ชี้ที่ตาราง) — resub 'มัก' ให้ค่าต่ำกว่าความจริง
เพราะมันไปสอบบนข้อสอบที่เพิ่งเห็นคำตอบมา มันเลย 'มองโลกในแง่ดี' บอกว่า error
น้อยกว่าจริง (ย้ำว่าชุดเดียวยังเหวี่ยงตาม seed ได้ บางชุดอาจไม่เป๊ะ เดี๋ยวข้อ b
ค่อยยืนยันด้วยการทำซ้ำ) ส่วน holdout กับ k-fold ที่วัดบนข้อมูลที่โมเดลไม่เคยเห็น
มักให้ค่าใกล้ความจริงกว่า นี่คือสัญญาณแรกของ bias ครับ"


>> (b) ทำซ้ำหลายชุด   (~1:20)   *** หัวใจของการจำลอง ***
(ชี้ที่จอ: histogram 'รูปรวม' ก่อน แล้วค่อยเลื่อนลงไป 'รูปแยกทีละวิธี')

"แต่ข้อมูลชุดเดียวยังตัดสินไม่ได้ เพราะแต่ละชุดที่สุ่มมาก็ให้ค่าไม่เท่ากันอยู่แล้ว
ผมเลยทำสิ่งที่การจำลองทำได้แต่ข้อมูลจริงทำยาก — สุ่มข้อมูลใหม่ซ้ำหลายร้อยชุด
รันทั้ง 3 วิธีทุกชุด แล้วเอาค่าที่ได้มาพล็อตเป็นการกระจาย (histogram)

ดูที่เส้นประคือค่าจริงนะครับ อ่านกราฟนี้ 2 มิติพร้อมกัน —
  * ตำแหน่งยอดการกระจาย เทียบกับเส้นประ = bias (เอนเอียงไหม)
  * ความกว้างของการกระจาย              = variance (แปรปรวนไหม)

  - แดง (resub)   เยื้องไปทางซ้ายจากเส้นประ  = เอนเอียง มองแง่ดีทุกครั้ง
  - เขียว (holdout) ยอดอยู่ใกล้เส้นประ แต่กระจายกว้างสุด = แปรปรวนสูง
  - น้ำเงิน (k-fold) แคบและอยู่คร่อมเส้นประพอดี = ทั้งไม่เอนเอียงและนิ่ง

(เลื่อนลงไปรูปแยก) และถ้าดูยากตอนซ้อนกัน ผมแยกให้ดูทีละวิธีบนแกนเดียวกัน
จะเห็นชัดเลยว่าแท่งเขียว (holdout) เตี้ยและกว้างสุด = เหวี่ยงสุด ส่วนน้ำเงิน
(k-fold) สูงชะลูดแคบ = นิ่งสุด เทียบความกว้างกันได้ตรงๆ เพราะสเกลเท่ากัน

และจุดที่ผมอยากเน้น — ในตารางนี้ทุกแถว MSE ~ bias^2 + variance ลงตัวเป๊ะ
นั่นแปลว่าการแยกว่า error มาจาก 'ความเอนเอียง' เท่าไร กับ 'ความแปรปรวน' เท่าไร
มันถูกต้องตามทฤษฎีจริงๆ ไม่ใช่บังเอิญครับ"


>> (c) เปลี่ยนสัดส่วนและ k   (~1:00)
(ชี้ที่จอ: กราฟแถบ +/- SD)

"ต่อมา ผมอยากรู้ว่าถ้าปรับวิธีแบ่งข้อมูล ความเที่ยงตรงจะเปลี่ยนไหม
แถบในกราฟคือช่วง +/-1 SD ยิ่งแคบยิ่งเที่ยงตรง เส้นประยังเป็นค่าจริงเหมือนเดิม

ฝั่ง k-fold พอเพิ่ม k แถบแคบลงเรื่อยๆ อย่างชัดเจน — เพราะเฉลี่ยจากหลายก้อน
ความแปรปรวนเลยถูกกดลง ส่วน holdout ต่อให้ปรับสัดส่วนยังไง แถบก็ยังกว้างและ
แกว่งกว่ามาก เพราะมันตัดสินจาก test ชุดเดียว ได้ชุดดีก็ดี ได้ชุดร้ายก็เพี้ยน

สรุปข้อนี้ — k-fold เที่ยงตรงกว่า holdout อย่างเห็นได้ชัด และยิ่ง k เยอะยิ่งนิ่ง"


>> (d) เปลี่ยน n และ noise   (~1:00)
(ชี้ที่จอ: กราฟ d)

"ข้อสุดท้ายของการจำลอง ผมปรับ 2 ปัจจัยที่อยู่นอกเหนือวิธีการ คือ
ปริมาณข้อมูล n และระดับ noise เพื่อดูว่าอะไรกระทบการประมาณบ้าง

  - ข้อมูลยิ่งเยอะ (n เพิ่ม)  = ทุกวิธียิ่งนิ่ง แถบแคบลงหมด (variance ลด)
                              เพราะข้อมูลเยอะ การสุ่มแต่ละครั้งต่างกันน้อยลง
  - noise ยิ่งมาก            = resub ยิ่งมองโลกแง่ดีหนักขึ้น เพราะมันไป
                              'จำ' noise ในชุดเทรนว่าเป็นสัญญาณ bias เลยยิ่งแย่

พูดง่ายๆ คือ variance แก้ได้ด้วยข้อมูลเยอะ แต่ bias ของ resub เป็นปัญหาเชิง
โครงสร้าง ยิ่ง noise มากยิ่งหลอกตัวเอง"

(สะพานเชื่อม) "ทั้ง 4 ข้อนี้คือแก่นของงาน — เราเห็นครบว่าใครเอนเอียง ใครแปรปรวน
เพราะการจำลองให้เรารู้ค่าจริง ทีนี้คำถามเดียวที่เหลือคือ — แล้วกับข้อมูลจริง
ที่เราไม่รู้ค่าจริง มันเป็นแบบเดียวกันไหม? ตรงนี้ผมเคยลงรายละเอียดไปแล้ว
รอบนี้ขอสรุปเร็วๆ ครับ"

  (4) ทดลองจริงด้วย Weka (สรุปเร็ว)   [ 7:00 - 9:00 ]
--------------------------------------------------------------------------------
(ชี้ที่จอ: section Weka)
  ** โน้ตผู้พูด: ส่วนนี้เคยนำเสนอละเอียดไปแล้ว พูดกระชับ ไล่เร็ว เอาแค่ข้อสรุป
     ไม่ต้องอ่านตัวเลขทุกตัว ชี้ที่ตารางแล้วชี้ประเด็นสำคัญพอ

"อันนี้เคยลงรายละเอียดไปแล้ว ผมขอไล่เร็วๆ นะครับ — ผมเอาแนวคิดเดียวกันไปลอง
บนข้อมูลจริง HeightWeight ทำนายน้ำหนักจากส่วนสูง ด้วย Linear Regression ใน Weka
วัด RMSE ซ้ำ 10 seed

(ชี้ตาราง Lab 1-3 รวดเดียว)
ผลตรงกับการจำลองทุกข้อครับ —
  - Holdout : SD เป็นรูปตัว U เที่ยงสุดตรงกลาง ปลายสองข้างพัง (test เหลือน้อย)
  - k-fold  : 10-fold เที่ยงตรงสุด SD แค่ 0.16 เทียบ holdout ที่ดีสุดยัง 0.72
              — เที่ยงกว่าราว 4.5 เท่า
  - เพิ่มข้อมูล 20->100 จุด : SD ลดฮวบทุกวิธี ตรงกับผลจำลองข้อ (d) เป๊ะ

(ชี้ที่จอ: ตาราง Reference RMSE 5.54)  *** จุดเดียวที่ขอเน้น ***
ที่อยากย้ำจุดเดียวคือ Lab 4 — ผมเทรนบนข้อมูลเต็มหมื่นตัว ได้ค่าจริง RMSE = 5.54
ใช้เป็นไม้บรรทัด แล้วสุ่มมาแค่ 20 ตัว: resub ได้ 5.29 (ต่ำกว่าจริง มองแง่ดี),
CV ได้ 5.80 (ใกล้จริงสุด ไม่เอนเอียง), holdout ได้ 6.62 SD สูงถึง 2.84 (เหวี่ยงสุด)

จุดสำคัญคือ — พอเราสร้างค่าจริงจากข้อมูลหมื่นตัวขึ้นมา เราเลยตอบเรื่อง bias
บนข้อมูลจริงได้ และผลออกมาตรงกับการจำลองที่ผมเน้นไปเมื่อกี้ทุกประการ"

  (5) สรุป + ปิด             [ 9:00 - 10:00 ]
--------------------------------------------------------------------------------
(ชี้ที่จอ: การ์ดสรุป 3 ใบ)

"สรุปทั้งงานเป็นประโยคเดียวครับ:
  - Resubstitution  — เที่ยงตรงแต่โกหก มองโลกแง่ดีเสมอ ห้ามใช้ประเมินจริง
  - Holdout         — ไม่เอนเอียงมาก แต่แปรปรวนสูง ขึ้นกับดวงว่าแบ่งได้ชุดไหน
  - k-fold CV       — ไม่เอนเอียง และเที่ยงตรงกว่า holdout -> ตัวเลือกที่ดีที่สุด

และที่ผมภูมิใจที่สุดคือ ข้อสรุปนี้ยืนยันได้สองทาง — ทั้งจากการจำลองที่ผมเน้น
วันนี้ และจากการทดลองจริงด้วย Weka ซึ่งให้คำตอบตรงกัน

ยิ่งข้อมูลน้อย noise มาก หรือโมเดลซับซ้อน การประมาณ E_out ยิ่งยาก — แต่ถ้าให้
เลือกวิธีเดียว เลือก k-fold ครับ ขอบคุณครับ"
