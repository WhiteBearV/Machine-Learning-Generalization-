  (1) เปิด + โจทย์            [ 0:00 - 1:00 ]
--------------------------------------------------------------------------------
(ชี้ที่จอ: หัวข้อสไลด์ + การ์ด 3 ใบหน้า intro)

"สวัสดีครับ วันนี้ผมจะพาไปดูคำถามง่ายๆ แต่สำคัญมากใน Machine Learning ครับ —
เวลาเราเทรนโมเดลเสร็จ เราจะรู้ได้ยังไงว่ามันเก่งจริงกับข้อมูลที่ไม่เคยเห็น?

ค่าที่เราอยากรู้เรียกว่า E_out หรือ error นอกกลุ่มตัวอย่าง ปัญหาคือเราวัดมัน
ตรงๆ ไม่ได้ ต้อง 'ประมาณ' เอา ซึ่งมีอยู่ 3 วิธีหลัก

คำถามของงานนี้คือ — 3 วิธีนั้น วิธีไหนประมาณได้ 'แม่น' และ 'นิ่ง' ที่สุด?
ผมจะตอบด้วย 2 ทาง คือการจำลอง และการทดลองจริงด้วยโปรแกรม Weka ครับ"


--------------------------------------------------------------------------------
  (2) แนวคิดหลัก             [ 1:00 - 2:30 ]
--------------------------------------------------------------------------------
(ชี้ที่จอ: การ์ด resub / holdout / kfold)

"เริ่มจาก 3 วิธีก่อนครับ
  - Resubstitution  — เทรนแล้ววัด error บนข้อมูลชุดเดิมที่เทรนไป
  - Holdout         — แบ่งข้อมูลเป็น train กับ test วัดบน test
  - k-fold CV       — แบ่งเป็น k ก้อน วนให้ทุกก้อนได้เป็น test แล้วเฉลี่ย

(ชี้ที่จอ: ตาราง Precision/Accuracy)

ทีนี้ 'แม่น' กับ 'นิ่ง' ที่ผมพูดถึง มันมีชื่อทางเทคนิคครับ
  - ความเที่ยงตรง (Precision) = ทำซ้ำแล้วได้ค่าใกล้เคียงกันไหม
        วัดด้วยค่าเบี่ยงเบนมาตรฐาน SD ยิ่งต่ำยิ่งเที่ยงตรง  ตรงกับ variance
  - ความแม่นยำ (Accuracy) = ค่าที่ได้ใกล้ความจริงไหม  ตรงกับ bias

เก็บสองคำนี้ไว้นะครับ เดี๋ยวเราจะใช้มันตัดสินทั้งงาน"


--------------------------------------------------------------------------------
  (3) ผลจากการจำลอง (a)-(d)  [ 2:30 - 5:45 ]
--------------------------------------------------------------------------------

>> (a) ข้อมูลชุดเดียว   (~0:45)
(ชี้ที่จอ: ตาราง a)

"ผมตั้งฟังก์ชันจริงเป็น sin(pi x) เพราะการจำลอง เรา 'รู้ความจริง' จึงวัด bias
ได้แม่น บนข้อมูลชุดเดียว จะเห็นว่า resub ให้ค่าต่ำกว่าความจริงเสมอ — มันมองโลก
ในแง่ดี เพราะไปสอบบนข้อสอบที่เคยเห็นแล้ว ส่วน holdout กับ k-fold ใกล้ความจริงกว่า"


>> (b) ทำซ้ำหลายชุด   (~1:00)
(ชี้ที่จอ: histogram + เส้นประค่าจริง)

"แต่ชุดเดียวยังตัดสินไม่ได้ ผมเลยสุ่มซ้ำหลายร้อยชุด แล้วพล็อตการกระจาย
ดูที่เส้นประคือค่าจริงนะครับ —
  - แดง (resub) เยื้องไปทางซ้าย         = เอนเอียง
  - เขียว (holdout) กระจายกว้างสุด      = แปรปรวนสูง
  - น้ำเงิน (k-fold) แคบและอยู่ตรงกลาง  = สมดุลดีสุด
และในตารางนี้ทุกแถว MSE ~ bias^2 + variance เป๊ะ ยืนยันว่าการแยกองค์ประกอบถูกต้อง"


>> (c) เปลี่ยนสัดส่วนและ k   (~0:45)
(ชี้ที่จอ: กราฟแถบ +/- SD)

"ทีนี้ลองปรับการแบ่งดู แถบคือช่วง +/-1 SD ยิ่งแคบยิ่งเที่ยงตรง เส้นประคือค่าจริง
ฝั่ง k-fold พอ k มากขึ้น แถบแคบลงชัดเจน ส่วน holdout แถบกว้างและแกว่งกว่ามาก —
k-fold เที่ยงตรงกว่า holdout อย่างเห็นได้ชัด"


>> (d) เปลี่ยน n และ noise   (~0:45)
(ชี้ที่จอ: กราฟ d)

"สุดท้ายปรับปริมาณข้อมูลกับ noise สรุปสั้นๆ —
ข้อมูลยิ่งเยอะ ทุกวิธียิ่งนิ่ง (variance ลด)
และ noise ยิ่งมาก resub ยิ่งมองโลกแง่ดีหนักขึ้น"

(สะพานเชื่อม) "ทั้งหมดนี้คือการจำลอง คำถามคือ — แล้วกับข้อมูลจริง
มันเป็นแบบเดียวกันไหม?"

  (4) ทดลองจริงด้วย Weka     [ 5:45 - 8:45 ]
--------------------------------------------------------------------------------
(ชี้ที่จอ: section Weka)

"ผมใช้ชุดข้อมูลจริง HeightWeight ทำนายน้ำหนักจากส่วนสูง ด้วย Linear Regression

>> Lab 1 : Holdout   (~0:40)
(ชี้ที่จอ: ตาราง Lab 1)

"เปลี่ยนสัดส่วน train ดู จะเห็น SD เป็นรูปตัว U — เที่ยงตรงสุดตรงกลางที่ 0.5
ส่วนปลายสองข้างพังหมด: ฝั่งเทรน 90% เหลือ test แค่ 2 จุดเลยเหวี่ยง
ฝั่งเทรน 10% เหลือ train แค่ 2 จุดโมเดลพังเลย RMSE พุ่งไป 21"


>> Lab 2 : k-fold   (~0:30)
(ชี้ที่จอ: ตาราง Lab 2)

"ฝั่ง cross-validation 10-fold เที่ยงตรงสุด SD แค่ 0.16 เทียบกับ holdout
ที่ดีสุดยังได้ 0.72 — CV เที่ยงตรงกว่าเกือบ 5 เท่า บนข้อมูลชุดเดียวกัน"


>> Lab 3 : เพิ่มข้อมูล   (~0:30)
(ชี้ที่จอ: ตาราง Lab 3)

"พอเพิ่มข้อมูลจาก 20 เป็น 100 จุด SD ลดฮวบทุกวิธี ค่าเฉลี่ยก็นิ่งขึ้น —
ยืนยันว่าข้อมูลเยอะช่วยทั้งความเที่ยงตรงและความแม่นยำ ตรงกับผลจำลองข้อ (d)"


>> Lab 4 : ไฮไลต์ *****   (~0:50)
(ชี้ที่จอ: ตาราง Reference RMSE 5.54)

"อันนี้เด็ดสุดครับ ผมเทรนบนข้อมูลเต็มหมื่นตัว ได้ค่าจริงคือ RMSE เท่ากับ 5.54
ใช้เป็นไม้บรรทัด แล้วสุ่มมาแค่ 20 ตัว เทียบ 3 วิธี
  - resub ได้ 5.29        — ต่ำกว่าความจริง มองโลกแง่ดีเหมือนเดิม
  - CV ได้ 5.80           — ใกล้ 5.54 สุดแบบไม่เอนเอียง
  - holdout ได้ 6.62 SD สูงถึง 2.84  — เหวี่ยงแรงสุด

จุดสำคัญคือ — บนข้อมูลจริงเราตอบเรื่อง bias ได้ เพราะเราสร้างค่าจริงขึ้นมา
จากข้อมูลหมื่นตัว และผลออกมาตรงกับการจำลองทุกประการ"


--------------------------------------------------------------------------------
  (5) สรุป + ปิด             [ 8:45 - 10:00 ]
--------------------------------------------------------------------------------
(ชี้ที่จอ: การ์ดสรุป 3 ใบ)

"สรุปทั้งงานเป็นประโยคเดียวครับ:
  - Resubstitution  — เที่ยงตรงแต่โกหก มองโลกแง่ดีเสมอ ห้ามใช้ประเมินจริง
  - Holdout         — ไม่เอนเอียงมาก แต่แปรปรวนสูง ขึ้นกับดวงว่าแบ่งได้ชุดไหน
  - k-fold CV       — ไม่เอนเอียง และเที่ยงตรงกว่า holdout -> ตัวเลือกที่ดีที่สุด

และที่ผมภูมิใจที่สุดคือ ข้อสรุปนี้ยืนยันได้สองทาง — ทั้งจากทฤษฎีที่จำลอง
และจากการทดลองจริงด้วย Weka ซึ่งให้คำตอบตรงกัน

ยิ่งข้อมูลน้อย noise มาก หรือโมเดลซับซ้อน การประมาณ E_out ยิ่งยาก — แต่ถ้าให้
เลือกวิธีเดียว เลือก k-fold ครับ ขอบคุณครับ"
